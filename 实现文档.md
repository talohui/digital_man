## 1. 项目整体架构

```
浏览器（React 18 前端，端口 5173）
  │
  ├──[HTTP POST]──► Fay 数字人框架（端口 5001）
  │                   ├── 调用阿里百炼 Qwen（LLM 问答）
  │                   ├── 调用阿里云 TTS（合成语音）
  │                   └── 查询 lingshan-rag（景区知识库）
  │
  ├──[WebSocket]──► Fay WS（端口 10003）
  │                   └── 推送：文字回复 + 音频 URL + 数字人状态
  │
  └──[HTTP POST]──► analytics-server（Spring Boot，端口 5002）
                      ├── 情感分析（KeywordSentimentAnalyzer）
                      ├── H2 数据库持久化
                      └──[HTTP GET] ◄── AdminDashboard / DataEase 大屏
```

**三条独立链路，互不耦合：**
- **对话链路**：HTTP 发消息 → Fay 处理 → WS 推回文字 + 音频
- **数字人链路**：WS 收到状态/音频 → Pixi + Live2D 渲染 + Web Audio 嘴型驱动
- **分析链路**：前端埋点 → analytics-server 存储 → 大屏轮询展示

### 3.2 Fay 通信层（`src/api/fay.ts`）

封装与 Fay 数字人框架的全部通信，对上层屏蔽协议细节。

#### HTTP 发送消息

```
POST http://127.0.0.1:5001/api/send
Content-Type: application/x-www-form-urlencoded
Body: data=<URL-encoded-JSON>
```

其中 JSON 结构：`{ "username": "User", "msg": "用户输入内容" }`

> **关键**：Fay 使用 `request.values.get('data')` 解析，必须用 x-www-form-urlencoded 格式，不能用 JSON 或 FormData。

#### WebSocket 连接

```
ws://127.0.0.1:10003
```

连接建立后，**必须立即发送注册帧**：
```json
{ "Username": "User" }
```

Fay 通过 Username 把 `panelReply`（数字人专属回复）路由到对应 socket。不发这一帧，Fay 只广播全局消息，收不到 `panelReply`。

#### 消息解析

Fay 推送的 JSON 格式不固定（不同版本字段位置有差异），`fay.ts` 提供两个解析工具函数：

| 函数 | 功能 |
|---|---|
| `extractAudioUrl(message)` | 依次尝试 `audioUrl`、`audio`、`url`、`panelReply.audio`，找到后拼成完整 URL |
| `extractRobotState(message)` | 解析 `robot` 字段，映射为 `normal / speaking / listening / thinking` 之一 |

文字回复提取优先级（`getMessageText`，在 store 中）：
1. `panelReply.content`（仅当 `panelReply.type === 'fay'` 时）
2. `text` → `message` → `content` → `answer` → `msg`
3. 排除 URL 和文件路径（防止把音频地址当文字显示）

### 3.3 全局状态管理（`src/store/useChatStore.ts`）

使用 Zustand 管理所有运行时状态，是整个应用的核心枢纽。

#### 状态字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `messages` | `ChatMessage[]` | 聊天历史，初始含欢迎词 |
| `wsStatus` | `idle/connecting/connected/disconnected/error` | WS 连接状态 |
| `inputText` | `string` | 输入框当前值 |
| `isSending` | `boolean` | HTTP 发送中标志（显示 loading） |
| `isRecording` | `boolean` | 录音中标志 |
| `lastError` | `string` | 最近一次错误提示 |
| `robotState` | `RobotState` | 数字人当前状态（驱动 motion） |
| `mouthOpen` | `number` | 嘴型开合度 0~1（音频振幅驱动） |
| `_lastSendTime` | `number` | 用户发消息的时间戳（计算 AI 响应时长） |

#### 核心 Action 逻辑

**`initializeConnection()`**：
- 幂等检查：socket 已 OPEN 则直接返回
- 调用 `connectFayWS`，回调绑定 `handleFayMessage`
- onOpen → 设 `connected`，调 `captureSessionStart()`
- onClose/onError → 设相应状态，调 `captureSessionEnd()`

**`handleFayMessage(message)`**：
- 提取文字 → `appendMessage('assistant', text)` + `captureAiReply()`
- 提取 robot 状态 → `setRobotState()`
- 提取音频 URL → `enqueueAudio(url)`

**`sendMessage(content)`**：
- 先 `appendMessage('user', content)` 显示气泡
- 记录 `_lastSendTime = Date.now()`
- `captureUserMessage(content)` 埋点（后端做情感分析）
- 设 `robotState: 'thinking'`
- 调 `sendTextToFay(content)` HTTP POST

**`enqueueAudio(url)`**：串行音频播放队列（核心设计，见下节）

### 3.4 Live2D 数字人（`src/lib/live2dManager.ts` + `src/components/Live2DStage.tsx`）

#### 架构分层

```
Live2DStage.tsx（React 组件）
  ├── 创建 PIXI.Application，绑定 canvas
  ├── 动态 import pixi-live2d-display/cubism4
  ├── 加载模型，调 registerModel() 注入单例
  └── 订阅 robotState / mouthOpen 变化 → 调 live2dManager

live2dManager.ts（纯 TS 单例）
  ├── registerModel(model)   存储模型引用
  ├── setMouthOpen(v)        写 ParamMouthOpenY 参数
  └── playMotionForState(s)  按状态切 motion group
```

单例设计使 `useChatStore` 和 `audioLipsync` 无需导入 React 组件即可驱动数字人。

#### Live2DStage 挂载流程

```tsx
// 1. 挂载 Pixi
const app = new PIXI.Application({
  view: canvas,
  resizeTo: canvas.parentElement,  // 自动跟随容器尺寸
  backgroundAlpha: 0,              // 透明背景
  antialias: true
})

// 2. 注入 PIXI 全局（pixi-live2d-display 0.4 要求）
(window as any).PIXI = PIXI

// 3. 动态加载 Cubism4 模块
import('pixi-live2d-display/cubism4').then(({ Live2DModel }) => {
  Live2DModel.registerTicker(PIXI.Ticker)  // 启动更新循环
  const model = await Live2DModel.from(modelUrl, { autoInteract: false })
  app.stage.addChild(model)

  // 4. 自适应缩放居中
  const scale = Math.min(w / baseW, h / baseH) * 0.9
  model.scale.set(scale)
  model.x = (w - baseW * scale) / 2
  model.y = (h - baseH * scale) / 2

  registerModel(model)
})
```

### 3.5 音频播放与嘴型同步（`src/lib/audioLipsync.ts`）

使用 **Web Audio API**（而非 `<audio>` 标签）的原因：必须接 `AnalyserNode` 才能实时读取振幅数据驱动嘴型。

#### 播放流程

```typescript
export async function playWithLipsync(url, onAmplitude): Promise<void> {
  // 1. fetch 音频 → decodeAudioData
  const buffer = await fetch(url).then(r => r.arrayBuffer())
                                 .then(b => audioCtx.decodeAudioData(b))

  // 2. 创建节点链：BufferSource → Analyser → Destination
  const source   = audioCtx.createBufferSource()
  const analyser = audioCtx.createAnalyser()
  source.buffer  = buffer
  source.connect(analyser)
  analyser.connect(audioCtx.destination)

  // 3. 每帧读取时域数据，计算 RMS 振幅
  const tick = () => {
    analyser.getByteTimeDomainData(data)
    let sum = 0
    for (const v of data) { const n = (v - 128) / 128; sum += n * n }
    const rms = Math.sqrt(sum / data.length)

    // 低通平滑（0.6/0.4 系数），防止嘴型抖动
    smoothed = smoothed * 0.6 + Math.min(1, rms * 3) * 0.4
    onAmplitude(smoothed)
    raf = requestAnimationFrame(tick)
  }

  // 4. 播放结束时 resolve，清理 RAF
  return new Promise(resolve => {
    source.onended = () => { cancelAnimationFrame(raf); onAmplitude(0); resolve() }
    source.start(); tick()
  })
}
```

#### 串行播放队列（在 `useChatStore` 中）

Fay 流式 TTS 可能分多段推送音频，必须按到达顺序串行播放：

```typescript
let audioChain: Promise<void> = Promise.resolve()
let pendingAudioCount = 0

enqueueAudio: (url) => {
  pendingAudioCount += 1
  set({ robotState: 'speaking' })
  audioChain = audioChain
    .then(() => playWithLipsync(url, (v) => set({ mouthOpen: v })))
    .catch(err => console.warn('音频播放失败:', err))
    .finally(() => {
      pendingAudioCount -= 1
      if (pendingAudioCount <= 0) {
        set({ mouthOpen: 0, robotState: 'normal' })
      }
    })
}
```

`audioChain` 是一个不断 `.then()` 延伸的 Promise 链，确保无论网络延迟如何，多段音频始终按顺序播放。

### 3.6 浏览器语音识别（`src/lib/browserAsr.ts`）

封装 `webkitSpeechRecognition`（Chrome/Edge 原生支持），Safari/Firefox 返回 `isSupported = false` 并给降级提示。

```typescript
const rec = new Ctor()
rec.lang            = 'zh-CN'
rec.interimResults  = false   // 只要最终结果，不要中间结果
rec.continuous      = false   // 说完一句即停

rec.onresult = (event) => {
  // 收集所有 isFinal 的片段拼接
  for (let i = event.resultIndex; i < event.results.length; i++) {
    if (event.results[i].isFinal) finalText += event.results[i][0].transcript
  }
}
rec.onend = () => {
  if (!stopped) return   // 自然结束但用户还没松手：等待 stop()
  opts.onFinal(finalText.trim())
}
```

**使用方式（`ChatPanel.tsx`）**：
- `onMouseDown / onTouchStart` → `beginRecord()` → `asr.start()`
- `onMouseUp / onTouchEnd / onMouseLeave` → `endRecord()` → `asr.stop()`
- `onFinal` 回调直接调用 `sendMessage(text)` 走完整发送流程

**错误处理**：
- `no-speech`：提示"没有识别到语音"
- `not-allowed`：提示"麦克风权限被拒绝"
- 不支持浏览器：提示使用 Chrome/Edge

### 3.7 行为埋点层（`src/lib/analytics.ts`）

**双通道并行发送**（fire-and-forget，不影响主流程）：

```
用户操作
  └──► capture(eventName, properties)
         ├── PostHog Cloud（VITE_POSTHOG_KEY 存在时启用）
         └── POST http://127.0.0.1:5002/api/events（始终发送）
                 Body: { event, properties, timestamp: ISO 8601 }
```

**已埋点的9类事件：**

| 事件常量 | 触发时机 | 关键字段 |
|---|---|---|
| `session_start` | WS onOpen | — |
| `session_end` | WS onClose/onError | `reason?` |
| `user_message` | 用户发送消息 | `content_text`（后端做情感分析）、`is_voice`、`content_length` |
| `ai_reply` | 收到 Fay 文字回复 | `latency_ms`、`reply_length` |
| `quick_ask` | 点击快捷问题按钮 | `question` |
| `voice_start` | 按下麦克风 | — |
| `voice_end` | 松开麦克风 | — |
| `audio_play_start` | 音频开始播放 | — |
| `audio_play_end` | 全部音频队列播完 | — |

> `content_text` 字段只发给 analytics-server 后端，由后端的 `KeywordSentimentAnalyzer` 做情感分析并存储结果，前端不做任何 NLP 处理。

### 4.2 情感分析策略模式

#### 接口设计

```java
// SentimentAnalyzer.java（Strategy 接口）
public interface SentimentAnalyzer {
    SentimentResult analyze(String text);
}

// SentimentResult.java（结果 Record）
public record SentimentResult(
    String sentiment,        // "positive" | "negative" | "neutral"
    double confidence,       // [0.0, 1.0]
    List<String> matched     // 命中的关键词列表（调试用）
) {}
```

#### 当前实现：`KeywordSentimentAnalyzer`

```java
@Component
@ConditionalOnProperty(name = "sentiment.provider", havingValue = "keyword", matchIfMissing = true)
public class KeywordSentimentAnalyzer implements SentimentAnalyzer {
    // 正面词（44个）：好、棒、赞、美、壮观、震撼、漂亮、精彩...
    // 负面词（36个）：差、糟、烂、贵、挤、失望、排队、麻烦...
    // 强度修饰词（8个）：非常、特别、超级、太、极其...
}
```

**评分算法：**
1. 遍历文本，统计命中的正/负面词
2. 若命中词前4个字符内含强度修饰词，该词权重 ×2
3. `posScore > negScore` → `"positive"`，`confidence = min(0.95, 0.5 + posScore/total × 0.5)`
4. `negScore > posScore` → `"negative"`，同上公式
5. 相等或无命中 → `"neutral"`，`confidence = 0.4`

#### 扩展方式

只需新建实现类并加 `@Primary` 注解，原有代码无需改动：

```java
// 未来接入阿里百炼 Qwen 情感分析
@Primary
@Component
@ConditionalOnProperty(name = "sentiment.provider", havingValue = "qwen")
public class QwenSentimentAnalyzer implements SentimentAnalyzer {
    // 调用 DashScope API，返回 SentimentResult
}
```

## 5. 完整数据流

### 用户发送消息的完整链路

```
用户输入文字 / 语音识别后
  │
  ├─[1] ChatPanel.tsx: handleSend() → sendMessage(content)
  │
  ├─[2] useChatStore:
  │      ├─ appendMessage('user', content)     → 气泡立即显示
  │      ├─ captureUserMessage(content)         → analytics.ts
  │      │     ├─ posthog.capture(...)          → PostHog Cloud（异步）
  │      │     └─ POST /api/events             → analytics-server（异步）
  │      │           └─ KeywordSentimentAnalyzer.analyze(content)
  │      │                 └─ 存入 analytics_events 表（含情感字段）
  │      └─ sendTextToFay(content)              → POST http://127.0.0.1:5001/api/send
  │
  └─[3] Fay 处理（异步，约 1~3s）：
         ├─ 调用 lingshan-rag 查知识库
         ├─ 调用 Qwen LLM 生成回答
         └─ 调用阿里云 TTS 合成语音
```

### Fay 回复的完整链路

```
Fay → WS 推送消息
  │
  ├─[4] useChatStore.handleFayMessage(message)
  │      ├─ getMessageText() 提取文字
  │      │     └─ appendMessage('assistant', text)  → 气泡显示
  │      │     └─ captureAiReply(text, _lastSendTime)  → 埋点时延
  │      ├─ extractRobotState() → setRobotState('speaking')
  │      │     └─ Live2DStage: useEffect → playMotionForState('speaking')
  │      └─ extractAudioUrl() → enqueueAudio(url)
  │
  └─[5] enqueueAudio(url) → audioChain.then(playWithLipsync)
         ├─ fetch(url) → Web Audio decode → play
         ├─ 每帧：AnalyserNode.getByteTimeDomainData → RMS → smoothed
         │     └─ set({ mouthOpen: smoothed })
         │           └─ Live2DStage: useEffect → setMouthOpen(v)
         │                 └─ coreModel.setParameterValueById('ParamMouthOpenY', v)
         └─ onended → mouthOpen=0, robotState='normal'
```

## 6. 关键技术决策说明

| 决策 | 方案 | 理由 |
|---|---|---|
| Live2D 渲染 | pixi-live2d-display@0.4 + pixi.js@6 | 社区最成熟方案；需 pixi@6，不兼容 pixi@7+ |
| Cubism Core 引入 | `<script>` CDN，先于 bundle | Cubism 协议要求独立引入，不能 npm 打包 |
| @pixi/* 子包 | 全部显式安装（36个） | npm 多次安装后 hoisting 失效，子包嵌套在 pixi.js/node_modules 中 |
| 音频播放 | Web Audio API | 需要 AnalyserNode 实时读取振幅，`<audio>` 标签无法做到 |
| 嘴型驱动 | RMS 振幅 + 低通平滑 | 不依赖 phoneme 模型，0延迟，效果可接受 |
| HTTP 格式 | x-www-form-urlencoded | Fay 服务端 `request.values.get('data')` 限制 |
| WS 注册帧 | `{"Username":"User"}` | Fay 用 Username 路由 panelReply，不发则收不到数字人专属回复 |
| ASR | webkitSpeechRecognition | demo 阶段零成本，Chrome/Edge 直接支持，不与 Fay 麦克风冲突 |
| 情感分析 | 后端 Strategy 模式 | 换模型只改一个 Bean，前端零改动 |
| 状态管理 | Zustand | 轻量，无 Provider 包裹，API 简洁 |
| 不用 StrictMode | — | 防止 WS / Pixi 双初始化（开发模式 useEffect 会执行两次） |
